This topic guides you through the standard practices utilized in [Twilio Video API](https://www.twilio.com/docs/video) and illustrates how they correspond with cocmd's API. The main purpose of this tutorial is to support you in migrating an existing application from Twilio to cocmd's video and audio APIs.

## Credentials

To access Twilio's API, you need an Account SID and Auth Token which can be obtained from `console.twilio.com`. cocmd uses Organization ID and API Key that you can get from the [Developer Portal](https://dev.cocmd.ai).

It's important to note that the credentials for both Twilio and cocmd are intended to be used only on the server-side.

| **Twilio**  | **cocmd**        |
| ----------- | --------------- |
| Account SID | Organization ID |
| API Secret  | API Key         |

## Rooms / Meeting

Twilio utilizes Rooms to establish connections between participants, whereas cocmd uses [Meetings](/api#/operations/create_meeting) for the same purpose.

| **Twilio** | **cocmd**   |
| ---------- | ---------- |
| Room       | Meeting    |
| Room SID   | Meeting ID |

## Participant Authentication

In Twilio, helper libraries are used to generate a participant access token.

```ts
const token = new AccessToken(twilioAccountSid, twilioApiKey, twilioApiSecret, {
  identity: identity,
});
token.addGrant(videoGrant);
console.log(token.toJwt());
```

In cocmd, you make a REST API call to [Add Participant API](/api/#/operations/add_participant) which returns `authToken`.

| **Twilio**   | **cocmd**  |
| ------------ | --------- |
| Access Token | Authtoken |
